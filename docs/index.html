Задание после теста (Time)

1. Откуда берётся переменная x? Какое у неё значение?

x — глобальная переменная, хранит начальный момент времени в миллисекундах (Unix timestamp). Она используется для расчёта прошедшего времени. Date.now() возвращает текущее время в момент каждой итерации.

2. Перепишите выражение без IIFE

Без IIFE:

console.time('a'); y = 2 ** 15; x = Date.now(); while (Date.now() - x < y) ; console.timeEnd('a');

Что такое IIFE? Это функция, которая выполняется сразу после создания, без отдельного вызова.

Отчёт по Webpack и Docker

1Инициализация проекта и установка зависимостей
npm init -y
npm i luxon
npm i -D webpack webpack-cli serve

Установили Webpack для сборки, luxon для работы с датой и временем.

2Настройка Webpack

Создан файл webpack.config.js с точкой входа src/index.js и выходным файлом dist/main.js.

const path = require('path'); module.exports = { entry: './src/index.js', output: { path: path.resolve(__dirname, 'dist'), filename: 'main.js' }, mode: 'development' };
3Создание кода с Luxon

В src/index.js написан код для отображения текущего времени с обновлением каждую секунду.

const { DateTime } = require('luxon'); const hh = document.createElement('h1'); document.body.appendChild(hh); setInterval(() => { hh.textContent = DateTime.local().setLocale('ru').toFormat('dd.Ll.Y HH:mm:ss'); }, 1000);
4Сборка проекта
npx webpack

Результат: файл dist/main.js (включает код + библиотеку luxon).

Работа с Docker

1Создание Dockerfile
FROM node:24-alpine WORKDIR /app COPY package*.json ./ RUN npm install COPY . . RUN npx webpack EXPOSE 8080 RUN npm install -g serve CMD ["serve", "-s", ".", "-l", "8080"]

Используется образ node:24-alpine для лёгкости и быстродействия.

2Сборка Docker-образа
docker build -t my-webpack-app .

Создан образ с именем my-webpack-app.

3Запуск контейнера
docker run -p 8080:8080 my-webpack-app

Приложение доступно по адресу http://localhost:8080.
